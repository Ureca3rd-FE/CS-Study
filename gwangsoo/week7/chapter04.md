# SQL과 NOSQL의 차이

## 개념

- **SQL (관계형 DB)**
  - 고정된 스키마 기반의 테이블에 데이터 저장
  - 테이블 간 관계로 데이터 무결성 유지
- **NoSQL (비관계형 DB)**
  - 스키마 유연한 문서/키-값/컬럼/그래프 형태
  - 관련 데이터를 하나의 문서에 담아 저장하는 방식(중복 저장 허용)

## 핵심 차이

- 스키마
  - SQL은 엄격한 스키마
  - NoSQL은 스키마 유연성
- 관계
  - SQL은 정규화로 중복 최소화
  - NoSQL은 중복 허용(빠른 읽기 위해 데이터 복제)
- 쿼리
  - SQL은 JOIN 등 복잡한 관계 쿼리 가능
  - NoSQL은 조인 개념 거의 없음
- 확장성
  - SQL은 주로 수직 확장(스케일 업)
    - NoSQL은 수평 확장(샤딩 등) 지원에 유리

## 장단점

### SQL 장점

- 명확한 스키마로 데이터 무결성 보장
- 정규화로 중복 최소화(일관성 유지 쉬움)
- 복잡한 관계형 쿼리(조인, 트랜잭션) 강점

### SQL 단점

- 스키마 변경이 번거로움(유연성 낮음)
- 복잡한 JOIN이 성능/복잡도 이슈 유발 가능
- 대규모 수평 확장 설계가 상대적으로 복잡

### NoSQL 장점

- 스키마 유연성(데이터 모델 빠르게 변경 가능)
- 읽기 성능 최적화 용이
- 수평 확장(대용량 분산 처리)에 적합

### NoSQL 단점

- 데이터 중복으로 업데이트 비용 증가
- 일관성 모델이 DB마다 달라 설계 주의 필요
- 관계 데이터 처리는 애플리케이션에서 직접 처리해야 함

## 언제 SQL을 선택할까?

- 데이터 간 관계가 많고 자주 변경되는 경우
- 강한 트랜잭션 보장이 필요한 경우
- 스키마가 명확하고 데이터 무결성이 중요한 비즈니스 로직

## 언제 NoSQL을 선택할까?

- 데이터 스키마가 불확실하거나 자주 변경될 가능성이 있는 경우
- 읽기 중심(조회 많고 쓰기 적음) 혹은 로그/이벤트 저장 등 고속 쓰기/읽기 필요시
- 대규모 수평 확장이 필요하고 샤딩/리플리카 기반 분산이 요구되는 경우
