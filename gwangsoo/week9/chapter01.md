# OSI 7 계층

![osi-model-7-layers](./assets/osi-model-7-layers.png)

OSI 7계층은 네트워크 통신 과정을 7단계로 나눈 모델로 **문제가 발생했을 때 어느 계층에서 오류가 생겼는지 빠르게 파악**하고 **각 계층을 독립적으로 관리하기 위해** 존재한다.

## 1) 물리 계층 (Physical Layer)

> 리피터, 케이블, 허브

- 데이터를 **전기 신호**(비트)로 변환해 전송하는 계층
- 장비 간 실제 물리적 연결을 담당
- 전송만 담당하며 데이터의 의미는 알지 못함

## 2) 데이터 링크 계층 (Data Link Layer)

> 브리지, 스위치

- **MAC 주소** 기반으로 프레임 단위 통신
- 오류 검출, 재전송, 흐름 제어 수행
- 물리 계층에서 발생할 수 있는 오류를 보정

### MAC 주소란?

MAC 주소는 **네트워크 장비(NIC)에 부여된 고유 식별자**로 데이터 링크 계층에서 통신 시 **장치 간을 구분하기 위해 사용되는 하드웨어 주소**이다.

- 전 세계에서 유일한 48비트 주소
- 보통 `00:1A:2B:3C:4D:5E` 같은 16진수 형태
- 스위치가 이 MAC 주소를 기반으로 프레임을 목적지에 전달함

## 3) 네트워크 계층 (Network Layer)

> 라우터, IP

- 데이터를 목적지까지 전달하는 역할
- **IP 주소** 기반 패킷 전달 및 라우팅
- 경로 선택, 흐름 제어, 오류 제어 수행

## 4) 전송 계층 (Transport Layer)

> TCP, UDP

- 컴퓨터 간 신뢰성 있는(또는 없는) 데이터 전송 담당

- **TCP**

  - 신뢰성 O
  - 연결 지향
  - 데이터 순서 보장, 재전송 있음
  - 웹 브라우징(HTTP/HTTPS), SSH, Telnet 같은 원격 접속

- **UDP**
  - 비신뢰성
  - 비연결성
  - 빠르며 실시간 통신에 적합
  - 동영상 스트리밍(YouTube, IPTV), 화상회의/VoIP(Zoom, Discord)

## 5) 세션 계층 (Session Layer)

> API, Socket

- 통신 세션(연결)을 생성, 유지, 종료하는 계층
- 애플리케이션 간의 논리적 연결 관리

## 6) 표현 계층 (Presentation Layer)

> JPEG, MPEG, 암호화/인코딩

- 데이터 표현 방식 변환
- 인코딩, 디코딩
- 압축, 암호화 수행
- 서로 다른 시스템이 같은 방식으로 데이터 해석할 수 있게 함

## 7) 응용 계층 (Application Layer)

> HTTP, FTP, DNS

- 사용자 프로그램이 직접 사용하는 계층
- 웹 서비스, 이메일, 데이터베이스 접근 등 제공
- OSI 모델 중 **사용자와 가장 가까운 계층**
