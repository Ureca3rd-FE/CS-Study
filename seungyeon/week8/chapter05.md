# 레디스 Redis

> Remote Dictionary Server, ~~원격 사전 서버~~

## 정의

오픈 소스 기반 **인 메모리** 데이터 저장소.

일반적인 DB와 달리 Disk가 아닌 `Memory(RAM)`에서 데이터를 처리 =`매우` 빠른 액세스 속도. (디스크 스캐닝이 필요 없기 때문에)

key-value 구조를 기반으로 데이터를 처리. =`key-value형 NoSQL`

## value 5가지

단순히 key-value 저장소인 게 아니라 다양한 데이터 구조를 지원. =복잡한 데이터를 손쉽게 저장하고 관리.

1. String(text, binary data)-512MB까지
2. set(String의 집합)
3. sorted set(set을 정렬한 상태)
4. Hash
5. List

## RAM은 휘발성이 아닌가?

처리된 데이터를 주기적으로 디스크 등에 동기화>데이터 유실 방지. (백업 과정)

#### 데이터 유실을 방지하기 위한 동기화 방식

1. 스냅샷(snapshot) <br>
   일정 주기마다 메모리 내 데이터를 디스크에 저장해 백업.

2. AOF(Append Only File)<br>
   모든 쓰기(쿼리) 작업을 기록하고 서버 셧다운 후 재시작 시 이를 기반으로 데이터 복구.

## 사용처

> 세션 관리: 세션 데이터를 메모리에 저장

- 사용자 인증 속도 향상
- 세션 만료 관리: Time to Live를 설정> 불필요한 데이터가 메모리 차지하지 않도록 자동 만료.
- 다중 서버 환경 지원: 로드밸런싱 환경에서 안정적으로 세션 관리

> 캐싱: 자주 조회되는 데이터를 메모리에 캐싱해 DB 부하 줄이기

- 읽기 부하 분산
- 복잡한 연산 속도 향상
- 만료 정책 지원: TTL로 오래된 거 치우고 최신 상태 유지

> 실시간 분석: 실시간 데이터 처리가 중요한 환경에서 빠른 데이터 조회와 집계 가능

- 대시보드 업데이트
- 실시간 통계 계산
- 이벤트 처리

> 메시지 큐: 실시간 메시지 전송과 수신을 간단하고 효율적으로

- Publish/Subscribe(Pub/Sub) 기능을 통해 메시지 브로커 역할을 수행. 실시간 소통 지원.
  - 이에 더해 빠름+발행 및 수신이 비동기적이란 특성 때문에 대용량 트래픽 상황에서 높은 성능 유지.
- 실시간 알림 시스템
- 채팅 어플리케이션
- 스트리밍 데이터 관리: Redis Stream 기능, 실시간으로 스트리밍 관리-여러 소비자에게 데이터 분배(확장성)
