# 로드 밸런싱 Load Balancing

> 둘 이상의 CPU or 저장장치와 같은 컴퓨터 자원들에게 작업을 나누는 것.

모든 트래픽을 1대의 서버가 감당하기엔 역부족. 대응 방안으로 Scale-up(하드웨어 성능 올리기), Scale-out(여러 대의 서버가 나눠서 일하도록)이 있음.

하드웨어 향상 비용은 비싸고, 서버가 여러 대 동작하는 편이 무중단 서비스를 제공하기 용이 > Scale-out이 효과적.

이때, 여러 서버에게 균등하게 트래픽을 분산시켜주는 것이 `로드 밸런싱`.

## 로드 밸런싱

분산식 웹 서비스. 클라이언트와 서버 사이에 배치되며 여러 서버에 요청을 나누어 줘 부하를 막는다. 웹 서버를 추가로 증설하면서 로드 밸런서로 관리해주면 부하 해결이 쉽다.

## 로드 밸런서가 서버를 선택하는 방식

- 라운드 로빈 Round Robin

  CPU 스케줄링의 라운드 로빈 방식 활용

- Least Connections

  연결 개수가 가장 적은 서버 선택(트래픽으로 인해 세션이 길어지는 경우 권장)

- Source

  사용자 IP를 해싱하여 분배(특정 사용자가 항상 같은 서버로 연결되도록 보장)

## 로드 밸런서 장애 대비

로드 밸런서에 문제가 생길 수 있기 때문에 로드 밸런서를 이중화하여 대비.

> Active 상태와 Passive 상태로
