# Chap02. 데브옵스 (DevOps)

## 1. 데브옵스란?
### 개념
- **개발(Development)**과 **운영(Operations)**을 통합하여 소프트웨어를 더 빠르고 안정적으로 전달하기 위한 문화/철학/방법론
- 개발팀: 기능 구현, 코드 작성 중심
- 운영팀: 배포, 서버 안정성, 장애 대응 중심

### 특징
#### 1. 협업과 커뮤니케이션 강화
- 개발자와 운영자가 같은 목표를 공유
- 책임을 분리하지 않고 공동 책임 구조

#### 2. 자동화 중심 (Automation)
- 수작업 최소화 -> 휴먼 에러 감소
- 대표적인 자동화 영역 
    - CI/CD (빌드, 테스트, 배포)
    - 인프라 프로비저닝
    - 모니터링 및 알림

#### 3. CI / CD (지속적 통합 / 지속적 배포)
- CI (Continuous Integration): 코드 변경 시 자동 빌드/테스트
- CD (Continuous Deployment/Delivery): 검증된 코드를 자동으로 배포

#### 4. 빠른 피드백과 반복 개선
- 배포 후 로그/에러를 즉시 확인
- 문제 발견 -> 빠른 수정 -> 다시 배포
- 짧은 개발 사이클 유지

#### 5. 안정성과 신뢰성 향상
- 자동 테스트 + 롤백 전략
- 장애 발생 시 빠른 복구
