# 7주차 🛅데이터베이스

## Chap04. SQL과 NOSQL의 차이

### #1. SQL (관계형 데이터베이스)

#### 개념

- 테이블, 행, 열 기반의 관계형 데이터 모델
- 스키마에 따라 데이터를 저장
- SQL(Structured Query Language)로 데이터를 조작

#### 특징

1. 스키마 구조가 미리 정의되어 있어야함 + 변경이 어려움
2. ACID 트랜잭션 지원

- 데이터 무결성과 일관성 보장
- 용어
  - 원자성(Atomicity): 트랜잭션의 모든 연산은 모두 수행됨 OR 모두 수행되지 않음
  - 일관성(Consistency): 트랜잭션 전후에 데이터 무결성 유지
  - 격리성(Isolation): 동시 실행되는 트랜잭션들은 서로 간섭 X
  - 지속성(Durability): 트랜잭션 성공 시, 결과는 영구적으로 저장

3. 데이터 조회 방식
   - 복잡한 관계를 조인으로 연결하여 조회
   - 중복 최소 + 데이터 무결성 높음
   - 조인 연산이 많아질수록 느려짐
4. 확장성: 수직적 확장(Scale-Up)으로 서버 성능 향상 중심
5. 복잡한 데이터 관계를 관리, 정확성/무결성이 중요한 서비스에 적합 (금융, 회원관리 등)

#### 예시

- MySQL, Oracle, PostgreSQL, ...

### #2. NoSQL (비관계형 데이터베이스)

#### 개념

- 문서, 키-값, 열 패밀리, 그래프 등 다양한 형태로 데이터 저장
- 대용량, 비정형 데이터 처리에 적합
- JOSN 기반의 문서 형태
- DB마다 고유한 API/쿼리 문법 존재

#### 특징

1. 스키마가 유연하며 필드 추가가 자유로움
2. BASE 원칙

- 데이터 가용성, 확장성 우선
- 용어
  - 기본적 가용성(Basically Available): 시스템은 항상 응답 가능해야함
  - 유연한 상태(Soft State): 데이터가 일시적으로 불일치할 수 있음
  - 최종 일관성(Eventual Consistency): 시간이 지나면 결국 데이터가 일관성에 도달함

3. 데이터 조회 방식
   - 데이터를 중복 저장하고, 내장 문서 구조를 통한 빠른 조회
   - 조인과 같은 연산 없이 한번에 조회 가능
   - 데이터가 많아지면 관리가 어려워짐
4. 확장성: 수평적 확장(Scale-Out)으로 서버를 여러대로 분산
5. 대량 데이터, 빠른 확장성이 필요한 서비스에 적합 (SNS, 로그 등)

#### 예시

- MongoDB, Cassandra, DynamoDB, ...
